## Configuration Parameter Guide

### Broker Settings

```json
"broker": "kite"                    // Broker adapter (currently only Kite supported)
"kite_api_key": "YOUR_KEY"          // Get from https://kite.trade/
"kite_token_file": "kite_token.json" // Generated by kite_login.py daily
"default_exchange": "NSE"           // NSE (equity), NFO (futures/options)
```

### Symbol Configuration

```json
"symbols": ["NIFTY24JANFUT"]        // List of instruments to trade
"timeframe": 30                     // Candle timeframe in minutes
"magic_number": 12345               // Unique bot identifier
```

**Timeframe Selection:**
- 1 min: Very active, noisy, for scalping
- 5 min: Active trading, good for options/equity
- 15 min: Balanced, good for BANKNIFTY
- 30 min: Conservative, good for NIFTY
- 60 min: Swing trading, fewer signals

### Risk Management

```json
"risk_percent": 1.0                 // Risk per trade (% of equity)
"reward_ratio": 2.0                 // Target profit (multiple of risk)
"max_daily_loss_percent": 3.0       // Stop trading after this loss
"max_drawdown_percent": 8.0         // Emergency stop level
```

**Risk Guidelines:**
- Beginners: 0.5-1% per trade
- Intermediate: 1-1.5% per trade
- Advanced: 1.5-2% per trade
- Options: 2-3% (due to premium decay)

### Technical Indicators

```json
"fast_ma_period": 10                // Fast moving average
"slow_ma_period": 21                // Slow moving average
"atr_period": 14                    // Average True Range period
"atr_multiplier": 2.0               // Stop loss distance (ATR multiple)
"rsi_period": 14                    // RSI calculation period
"rsi_overbought": 70                // Overbought threshold
"rsi_oversold": 30                  // Oversold threshold
```

**Indicator Tuning:**
- Faster MA = More signals, more noise
- Slower MA = Fewer signals, better quality
- Higher ATR multiplier = Wider stops, fewer stop-outs
- Lower ATR multiplier = Tighter stops, more stop-outs

### Position Management

```json
"use_split_orders": true            // Split exits for profit booking
"num_positions": 3                  // Number of split positions
"tp_levels": [1.0, 1.5, 2.5]       // Take profit levels (risk multiples)
"partial_close_percent": [40, 30, 30] // % to close at each TP
```

**Split Order Strategy:**
- First TP: Book quick profits (40-50%)
- Second TP: Let winners run (30%)
- Third TP: Capture big moves (20-30%)

### Trading Hours

```json
"trading_hours": {
  "start": "09:15",                 // Market open (IST)
  "end": "15:30"                    // Market close (IST)
}
"product_type": "MIS"               // MIS (intraday) or NRML (delivery)
```

**Product Types:**
- **MIS:** Margin Intraday Square-off
  - Auto-squared at 3:20 PM
  - Higher leverage (5-10x)
  - Lower margin requirement
  - Recommended for intraday

- **NRML:** Normal (Delivery/Carry Forward)
  - Can hold overnight
  - Lower leverage (2-3x)
  - Higher margin requirement
  - Not recommended for options

### Advanced Features

```json
"use_adaptive_risk": true           // Adjust risk based on performance
"ml_enabled": false                 // Machine learning signal filtering
"use_volume_filter": true           // Filter by volume
"use_trend_detection": true         // Use trend detection engine
```

**Feature Recommendations:**
- Adaptive Risk: Always enable (adjusts to market conditions)
- ML: Optional (requires training data)
- Volume Filter: Enable for equity/options (critical for liquidity)
- Trend Detection: Enable (avoids choppy markets)

### Volume and Trend Filters

```json
"min_volume_ratio": 1.2             // Minimum volume vs average
"volume_ma_period": 20              // Volume MA period
"adx_period": 14                    // ADX calculation period
"adx_threshold": 25                 // Minimum trend strength
```

**Filter Tuning:**
- Higher volume ratio = More selective (fewer trades)
- Higher ADX threshold = Stronger trends only (fewer trades)
- Lower thresholds = More trades, lower quality

### Position Limits

```json
"max_positions": 2                  // Maximum concurrent positions
"max_trades_per_day": 5             // Daily trade limit
```

**Limit Guidelines:**
- Futures: 1-2 positions (high capital requirement)
- Equity: 3-5 positions (diversification)
- Options: 4-6 positions (lower capital per trade)

## Quick Start Guide

### Step 1: Choose Your Configuration

1. **Beginner?** Start with `config_nifty_futures.json`
2. **Small capital (<â‚¹1L)?** Use `config_equity_intraday.json`
3. **Experienced?** Try `config_banknifty_futures.json`
4. **Options trader?** Use `config_options_trading.json`

### Step 2: Update Configuration

```bash
# Copy the example configuration
cp config_nifty_futures.json my_config.json

# Edit with your settings
# - Add your Kite API key
# - Update symbols to current expiry (for futures/options)
# - Adjust risk parameters to your comfort level
```

### Step 3: Authenticate

```bash
# Run the login script daily
python kite_login.py

# This generates kite_token.json with your access token
```

### Step 4: Test with Paper Trading

```bash
# Use paper trading mode first
cp config_paper_trading.json my_config.json

# Edit my_config.json and set:
"paper_trading": true

# Run the bot
python run_bot.py --config my_config.json
```

### Step 5: Go Live

```bash
# After testing, disable paper trading
"paper_trading": false

# Start with small position sizes
"risk_percent": 0.5  # Start conservative

# Run the bot
python run_bot.py --config my_config.json
```

## Performance Optimization Tips

### 1. Backtest First
- Test configuration on historical data
- Verify win rate and profit factor
- Adjust parameters based on results

### 2. Start Conservative
- Begin with lower risk (0.5%)
- Use wider stops initially
- Reduce position limits

### 3. Monitor and Adjust
- Track daily performance
- Adjust risk based on results
- Fine-tune indicator parameters

### 4. Use Filters
- Enable volume filter for liquidity
- Use ADX for trend strength
- Avoid choppy market conditions

### 5. Respect Market Hours
- Best liquidity: 9:30-11:00 AM, 2:00-3:15 PM
- Avoid: 12:00-1:00 PM (lunch hour)
- Most volatile: First 30 minutes

## Troubleshooting

### No Signals Generated
- Check if market is open
- Verify ADX threshold (lower it)
- Check volume filter (reduce min_volume_ratio)
- Verify symbol names are correct

### Too Many Signals
- Increase ADX threshold
- Increase volume filter
- Use longer timeframe
- Tighten indicator parameters

### Frequent Stop-Outs
- Increase ATR multiplier
- Use longer timeframe
- Reduce position size
- Check if market is choppy

### Authentication Errors
- Run kite_login.py daily
- Check API key is correct
- Verify token file exists
- Check internet connection
