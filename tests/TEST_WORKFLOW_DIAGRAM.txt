╔══════════════════════════════════════════════════════════════════════════════╗
║                          TYPICAL TEST DAY WORKFLOW                           ║
╚══════════════════════════════════════════════════════════════════════════════╝

Morning:
--------
1. Open TEST_EXECUTION_TRACKING.csv
2. Identify tests not yet executed
3. Review TEST_SCENARIOS_DETAILED.md for those tests
4. Set up test environment

During Testing:
--------------
1. Execute test following TEST_SCENARIOS_DETAILED.md
2. Record results in TEST_EXECUTION_TRACKING.csv
3. If test fails, create entry in DEFECT_TRACKING.csv
4. Take screenshots and capture logs
5. Move to next test

End of Day:
----------
1. Update TEST_EXECUTION_TRACKING.csv with all results
2. Review DEFECT_TRACKING.csv for new issues
3. Calculate progress (tests executed / 100)
4. Communicate findings to team

End of Cycle:
------------
1. Verify all 100 tests executed
2. Check TEST_COVERAGE_MATRIX.csv for gaps
3. Complete TEST_RESULTS_SUMMARY.md
4. Present findings and get sign-off

╔══════════════════════════════════════════════════════════════════════════════╗
║                          DEFECT WORKFLOW DETAIL                              ║
╚══════════════════════════════════════════════════════════════════════════════╝

1. DEFECT FOUND
   │
   ├─→ Create entry in DEFECT_TRACKING.csv
   │   - Assign unique Defect ID (DEF001, DEF002, etc.)
   │   - Link to Test ID that found it
   │   - Set Severity (Critical/High/Medium/Low)
   │   - Set Priority
   │   - Describe issue
   │   - Add steps to reproduce
   │
   ├─→ Update TEST_EXECUTION_TRACKING.csv
   │   - Mark test as "Fail"
   │   - Add Defect ID reference
   │   - Set "Retest Required" to Yes
   │
   └─→ Notify developer

2. DEFECT FIXED
   │
   ├─→ Update DEFECT_TRACKING.csv
   │   - Change Status to "Fixed"
   │   - Add Fixed Date
   │   - Add Fix Version
   │
   └─→ Schedule retest

3. RETEST
   │
   ├─→ Re-execute test from TEST_CASES_MASTER.csv
   │
   ├─→ Update TEST_EXECUTION_TRACKING.csv
   │   - Add Retest Date
   │   - Update Retest Status
   │
   └─→ Update DEFECT_TRACKING.csv
       │
       ├─→ If PASS: Close defect
       │
       └─→ If FAIL: Reopen defect, add new info

For TEST_CASES_MASTER.csv:
--------------------------
1. Open Excel
2. File → Open → Browse
3. Select TEST_CASES_MASTER.csv
4. Data imports with 11 columns
5. Apply filters to header row
6. Freeze top row for easy scrolling
7. Save as .xlsx if needed

For TEST_EXECUTION_TRACKING.csv:
--------------------------------
1. Open Excel
2. File → Open → Browse
3. Select TEST_EXECUTION_TRACKING.csv
4. Data imports with 13 columns
5. Use for daily test tracking
6. Fill in as you execute tests
7. Save frequently

For DEFECT_TRACKING.csv:
-----------------------
1. Open Excel
2. File → Open → Browse
3. Select DEFECT_TRACKING.csv
4. Data imports with 19 columns
5. Add new row for each defect
6. Track through to resolution
7. Save frequently

For TEST_COVERAGE_MATRIX.csv:
-----------------------------
1. Open Excel
2. File → Open → Browse
3. Select TEST_COVERAGE_MATRIX.csv
4. Data imports with 4 columns
5. Review for coverage gaps
6. Use for planning
7. Reference only (no updates needed)
